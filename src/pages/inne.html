GIT

do czego służy GIT?

GIT służy do budowania historii projektu, umożliwia wracanie do wcześniejszych wersji, pozwala na zbudowanie historii zmian, pozwala na jednoczesna prace różnych osób na tym samym projekcie. Działa tylko i wyłącznie lokalnie, ale może zostać połączony z serwerem zewnętrznym (np.github).

jak używać GITa

Przede wszystkim należy zacząc od instalacji i ednorazowej konfiguracji, nastepnie przy pomocy poleceń status, add, commit możemy kolejno weryfikować co mamy w repozytorium gita, dodawać pliki oraz commitować czyli zapisywać kolejne wersje. Na serwer zewnetrzny polecenia ładujemy za pomącą komendy git push.
Jak Cie widzą, tak cię piszą. Twoja strona, twoje repozytorium - Twoją wizytówką.

Giedy powinniśmy commitować?

Commitujemy po zakończonym etapie, tzn np. nowo utworzonej sekcji, artykul, tabeli itp. Na początku na pewno warto commitować częściej niż rzadziej, żeby wyrobić sobie nawyk. Jak doświadczenie uczy przezorny zawsze ubezpieczony, więc lepiej mieć więcej wersji niż mniej. Pamietajmy jednak, że repozytorium GITa bedzie w przyszłości naszą wizytówką. Warto zatem, zadbać o to, żeby nasze commity miały sens:)
